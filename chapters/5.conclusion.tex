%================ Section 5.1: Summary ==================
\section{สรุปผลการดำเนินงาน}
\label{sec:summary}

\noindent
โครงงาน GymMate ได้ออกแบบและพัฒนาระบบต้นแบบสำหรับสนับสนุนการออกกำลังกายของนักศึกษามหาวิทยาลัยเชียงใหม่ในหลายมิติ โดยมีผลลัพธ์สำคัญดังนี้

\begin{itemize}
  \item \textbf{ระบบ Workout Program Planner} — รองรับทั้ง Preset Plan และ Custom Plan พร้อมแนวทาง Goal \& Approach เช่น Push–Pull–Legs, Upper–Lower และ Total Body ผู้ใช้สามารถดำเนินการออกกำลังกายตามปฏิทินและบันทึกผลลัพธ์ได้จริง
  \item \textbf{ระบบ Flex Substitute} — ใช้อัลกอริทึม Cosine Similarity เพื่อแนะนำท่าทดแทนที่เน้นกล้ามเนื้อกลุ่มเดิม โดยผลลัพธ์ถูกคำนวณและเก็บไว้ล่วงหน้า (pre-computed) เพื่อประสิทธิภาพสูงสุด
  \item \textbf{Interactive Floorplan} — แผนผังยิมแบบ Interactive ให้ผู้ใช้สำรวจตำแหน่งเครื่องและสถานะได้ พร้อมระบบ Editor ฝั่ง Admin
  \item \textbf{Gamification Layer} — ระบบ XP, Level, Badge, Streak และ Leaderboard ช่วยเพิ่มแรงจูงใจและความต่อเนื่องในการออกกำลังกาย
  \item \textbf{Admin \& Trainer Dashboard} — Web CMS ครบถ้วนสำหรับจัดการ exercise, program, floorplan และผู้ใช้งาน
  \item \textbf{Clean Architecture} — Backend ออกแบบตาม Clean Architecture ด้วย Go + Gin + GORM + PostgreSQL ทำให้ unit test ได้โดยไม่ต้องพึ่ง database จริง และผ่าน CI pipeline อัตโนมัติ
\end{itemize}

%================ Section 5.2: Problems ==================
\section{ปัญหาและอุปสรรคที่พบ}
\label{sec:problems}

\begin{itemize}
  \item \textbf{ข้อจำกัดด้านเวลา} — ระยะเวลาในการพัฒนาไม่เพียงพอสำหรับการพัฒนาระบบให้สมบูรณ์และทดสอบกับผู้ใช้งานจริงในวงกว้าง
  \item \textbf{ขาดข้อมูลจากการใช้งานจริง (Real-world Data)} — ระบบบางส่วน เช่น Flex Substitute และ Gamification ยังไม่สามารถปรับให้เหมาะสมกับผู้ใช้จริงได้อย่างเต็มที่ เนื่องจากไม่มีข้อมูลพฤติกรรมผู้ใช้จริง
  \item \textbf{ข้อจำกัดด้านฮาร์ดแวร์} — ระบบไม่ได้เชื่อมต่อกับอุปกรณ์ฟิตเนสจริง ทำให้ไม่สามารถเก็บข้อมูลแบบ real-time เช่น การใช้งานเครื่องหรือ biometric data
  \item \textbf{ความซับซ้อนของการออกแบบระบบ} — การออกแบบระบบที่มีหลายฟีเจอร์ เช่น Planner, Gamification และ Floorplan ทำให้ต้องใช้เวลาในการออกแบบโครงสร้างระบบและฐานข้อมูลค่อนข้างมาก
  \item \textbf{ข้อจำกัดด้านการทดสอบ} — ยังไม่มีการทดสอบในระดับ production หรือ load test อย่างเต็มรูปแบบ
\end{itemize}

%================ Section 5.3: Future Work ==================
\section{ข้อเสนอแนะและแนวทางพัฒนาต่อ}
\label{sec:future}

\noindent
เพื่อพัฒนา GymMate ให้สามารถนำไปใช้งานจริงได้อย่างมีประสิทธิภาพ ควรมีการปรับปรุงในด้านต่างๆ ดังนี้

\begin{itemize}
  \item \textbf{พัฒนา Personalization และ AI Recommendation} — ใช้ Machine Learning เพื่อแนะนำโปรแกรมการออกกำลังกายที่เหมาะสมกับผู้ใช้แต่ละคนจากพฤติกรรมการใช้งานจริง
  \item \textbf{เชื่อมต่อกับอุปกรณ์ IoT หรือ Fitness Tracker} — เช่น Smartwatch หรือเครื่องออกกำลังกาย เพื่อเก็บข้อมูลแบบ real-time และเพิ่มความแม่นยำของระบบ
  \item \textbf{ปรับปรุงระบบ Flex Substitute} — ให้สามารถแนะนำท่าทางได้แม่นยำมากขึ้น โดยอิงจากข้อมูลผู้ใช้จริงและบริบทของยิม เช่น ความหนาแน่นของผู้ใช้งาน
  \item \textbf{พัฒนา Gamification ให้หลากหลายขึ้น} — เช่น ระบบ Challenge, Group Competition หรือ Seasonal Event เพื่อเพิ่ม engagement
  \item \textbf{รองรับระบบแบบ Real-time} — เช่น การแสดงสถานะเครื่องว่าง/ไม่ว่างแบบ real-time ผ่าน WebSocket หรือ Server-Sent Events
  \item \textbf{ขยายระบบเป็น Microservices Architecture} — เพื่อรองรับการใช้งานในระดับใหญ่และเพิ่ม scalability
  \item \textbf{ปรับปรุงด้าน UI/UX อย่างต่อเนื่อง} — โดยอิงจาก feedback ของผู้ใช้งานจริง เพื่อให้ใช้งานง่ายและตอบโจทย์มากขึ้น
\end{itemize}

คณะผู้จัดทำหวังเป็นอย่างยิ่งว่าโครงงาน GymMate จะเป็นเครื่องมือที่ช่วยส่งเสริมการออกกำลังกายให้มีประสิทธิภาพมากยิ่งขึ้น และสามารถนำไปต่อยอดให้เกิดประโยชน์ต่อผู้ใช้งานในอนาคต
